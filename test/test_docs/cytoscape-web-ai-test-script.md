# **Cytoscape Web — End-to-End Feature Test Script (for AI Agent)**

Cytoscape Web v1.0.4  
*Last updated*: 2025-09-18

**Purpose**

* Guide an AI agent through a complete, thorough, UI-driven test of Cytoscape Web.  
* Validate that all major features function as documented, with special emphasis on Styles, Tables, Search, Network View tools, and hierarchical “Cell View”.

**Preflight & Environment**

* Browser: Use latest Chrome/Firefox/Edge. If using Brave, disable Shields.  
* NDEx: Ensure an NDEx account exists (or create one) for tests involving “My Networks/Workspaces” and “Save to NDEx”.  
* Local Desktop (for “Open in Cytoscape”): Install Cytoscape Desktop 3.10+; ensure it’s running and the ‘cx support core-app’ is up to date.  
* Data: Prepare small sample files:  
  * CX2 networks (1 small; 1 medium)  
  * CSV/TSV/TXT edge list (with: source, target, interaction; plus extra numeric & categorical columns)  
  * Node-only list (single column) to verify node-only import  
  * CSV table to import to nodes and a separate CSV table to import to edges  
  * One “invalid/unsupported” file (e.g., .sif) to verify graceful error handling  
* Permissions: Have a test NDEx account with at least one writable network and workspace to test overwrite vs copy.

**Pass/Fail & Logging Guidance**

* For each step, record: PASS/FAIL, screenshot (if helpful), error messages, and environment snapshot (browser, OS, Cytoscape Desktop version).  
* Where applicable, also log NDEx UUIDs (networks/workspaces) used and any URLs generated by Share Network.  
* Capture before/after counts of nodes, edges, and table rows when testing Merge, Edit, and Imports.

**0\) Global Smoke Checks (Landing Page)**

### **0.1 Load Cytoscape Web. Confirm menu bar, Workspace Panel (left), Network View Panel (center), Table Panel (bottom) are visible. Resize and collapse panels; verify the UI remembers sizes while active.**

### **0.2 Verify the Sign In (NDEx) button is present on the right side of the menu bar.**

### **0.3 Verify the Search Tool is visible; open its settings and confirm default values: OR, Nodes, Exact match ON.**

## **1\) Data Menu**

### **1.1 Open Network(s) from NDEx**

a) Click Data → Open Network(s) from NDEx.

b) In SEARCH NDEx, search a common term (e.g., “galFiltered”).

c) Select one or more importable networks (not greyed out) and OPEN NETWORK(S).

d) If signed in, switch to MY NETWORKS and import from own collection.

e) Validate: networks appear in Workspace; current network renders; counts match NDEx metadata.

### **1.2 Open Sample Networks**

a) Data → Open Sample Networks.

b) Import each sample; confirm each loads and renders.

c) Identify which sample is hierarchical (tree+cell view available) for later tests.

### **1.3 Open Workspace from NDEx**

a) Sign in (if needed). Data → Open Workspace from NDEx.

b) Select an existing workspace; OPEN WORKSPACE.

c) Validate: all networks and prior app/service-app registrations load; the selected network renders.

### **1.4 Import → Network from File**

a) Import CX2 (.cx2); confirm expected nodes/edges and attributes.

b) Import delimited text (.csv/.tsv/.txt) with source/target/interaction; map columns; preview updates with changes.

c) Import a node-only list (single column); verify a node-only network is created.

d) Use Advanced Settings: “Transfer first line as column names” and “Start Import Row”; verify effects.

e) Negative test: Attempt to import an unsupported format (e.g., .sif); expect informative error/disable.

### **1.5 Import → Table from File**

a) With a network loaded, import a CSV table into Node table; set network Key column and Data Key column correctly.

b) Repeat for Edge table import.

c) Validate row counts mapped; mismatches are reported; data types (String/Boolean/Integer/Double/List) are honored.

### **1.6 Save to NDEx**

a) Data → Save Network to NDEx (overwrite). Expect success only when user owns/has edit permission.

b) Data → Save Copy to NDEx; confirm a new network in NDEx “My Networks” with distinct UUID.

c) Data → Save Workspace; then modify something and Data → Save Workspace As…; confirm both appear under “My Workspaces”.

### **1.7 Download Current Network (.cx2)**

a) Data → Download Current Network (.cx2). Save and then re-import to verify integrity.

### **1.8 Export → Network to Image**

a) Export PNG; verify scale/background options.

b) Export PDF; verify paper size options; open file to confirm vector output.

c) Export SVG; verify opens in vector editor.

d) Toggle “Export full network image” on/off; compare full vs viewport-only outputs.

### **1.9 Remove → Remove Networks**

a) Remove the current network and also remove multiple selections via Workspace; confirm removal from Workspace list.

### **1.10 Clear Local Database**

a) Data → Clear Local Database. Confirm the app resets (networks, apps/service apps, and local settings cleared).

b) Reopen app and confirm a clean slate.

## **2\) Edit Menu**

### **2.1 Delete Selection**

a) Select a subset of nodes and/or edges; Edit → Delete Nodes/Edges (as offered).

b) Validate the Table Panel and counts update; confirm deletion persists to NDEx on save/copy.

### **2.2 Undo/Redo**

a) Perform a few operations, such as moving a node, changing a layout, or modifying a default mapping

b) Edit → Undo to confirm the performed operations are reversed successfully.

c) Edit → Redo to confirm the operations are executed again correctly.

## **3\) Layout Menu**

### **3.1 One-click Layouts**

a) Apply each: COSE, Cosmos, Force-directed (gForce), Circular, Concentric, Grid, Radial, Hierarchical (DAGRE).

b) Validate geometry changes; labels remain legible; no rendering glitches or stuck nodes.

### **3.2 Layout → Settings…**

a) Pick a layout and tweak parameters; apply to confirm effect.

b) Set a Default Layout in settings.

c) Use Network View tool “Apply Default Layout” and confirm it uses the chosen default.

d) Apply to small and medium networks; verify performance and stability.

## **4\) Analysis Menu**

### **4.1 LLM Query**

a) On a map-like network whose nodes are “communities of genes”, run Analysis → Run LLM Query.

b) If no API key set, confirm a prompt to configure appears.

c) Configure via Analysis → LLM Query Options (model choice, API key, prompt template). Key should be stored locally & used securely.

d) Run again; validate output panel appears with reasonable annotations. Record response time and any errors.

e) Negative tests: run LLM Query on a standard (non-map) network; expect a clear “not supported” warning.

f) Switch models in Options; re-run to confirm model change takes effect.

## **5\) Tools Menu**

### **5.1 Merge Networks**

a) Tools → Merge Networks.

b) Basic operations: run UNION, INTERSECTION, DIFFERENCE using 2–3 networks; verify a new merged network is created each time.

c) Advanced Options:

\- Set Merged Network Name.

\- Matching columns: change matching attribute and re-run; validate node merge behavior.

\- Column merge mapping: set names and data types for Nodes/Edges/Network in the merge table; verify results.

\- Enable “merge nodes/edges within the same network”: run with a single network; verify internal deduplication.

\- “Merge only nodes and ignore edges” (Intersection only): verify node-only output.

d) Save merged outputs to NDEx and verify provenance/metadata are sensible.

## **6\) Apps Menu**

### **6.1 Manage Apps**

a) Apps → Manage Apps; toggle the example App(s) on/off; ensure UI elements appear/disappear accordingly.

### **6.2 Example App**

a) Apps → Simple Dialog Example; expect a “Hello Cytoscape World\!” dialog; close it.

### **6.3 Service Apps**

a) In Manage Apps, register an External Service URL (e.g., Update Tables example); Add and enable.

b) Run it from Apps → Example Service App → Update tables example; verify it modifies/augments table data.

### **6.4 Community Service Apps**

a) Register and enable: g:Profiler Enrichment and IQuery Enrichment endpoints.

b) For a network with a node column containing gene symbols list(s), run enrichment; verify results appear and are actionable.

### **6.5 Persistence**

a) Save Workspace; reload Cytoscape Web and open that workspace; confirm Apps and Service Apps are restored.

b) Clear Local Database; confirm Apps/Service Apps are cleared.

## **7\) Help Menu**

### **7.1 About, Manual, Code Repository, Report a Bug, Citation**

a) Each item should open the correct page in a new tab/window; verify links resolve.

b) Confirm the Citation text is present and correct.

## **8\) Sign In Button (NDEx)**

### **8.1 Sign In Flow**

a) Click the Sign In icon; complete sign in (or Sign in with Google).

b) If first time linking, complete email verification; re-test sign in.

### **8.2 Signed-in Features**

a) Data → Open Network(s) from NDEx → MY NETWORKS tab is available and correct.

b) Data → Open Workspace from NDEx shows your workspaces.

c) Data → Save Network/Save Copy/Save Workspace/Save Workspace As… succeed.

### **8.3 Sign Out**

a) Sign out (if available) or clear local database; verify access is revoked until sign in again.

## **9\) Search Tool (Menu Bar)**

### **9.1 Basic Search**

a) Search for a term present in node names; verify matched nodes highlight.

b) Change scope to Edges; search for a term in an edge column; verify matched edges highlight.

### **9.2 Settings**

a) Toggle OR vs AND; verify multi-term behavior changes.

b) Toggle Exact Match on/off; verify wildcard behavior (with and without partial terms).

c) Case-insensitivity: search for a term in mixed case; results should be identical.

### **9.3 No Results**

a) Search a nonsense term; confirm a “no matches” state is clearly communicated.

## **10\) Workspace Panel**

### **10.1 Multi-network Handling**

a) Load 3–4 networks; click each to make it current; confirm switching updates the view and table.

### **10.2 Provenance Badges**

a) Confirm NDEx badge for NDEx-sourced networks; file badge for file imports.

### **10.3 Panel Behavior**

a) Resize, collapse/expand; verify behavior is smooth and stateful during the session.

## **11\) Network View Panel**

### **11.1 Network View Tools (bottom-right)**

a) Apply Default Layout → uses default from Layout Settings.

b) Fit Network to Window → network fits fully in panel.

c) Open in Cytoscape → with desktop running, network opens in Cytoscape Desktop (default port 1234).

d) Share Network → URL is copied to clipboard; open it in a fresh tab to verify it loads the same network/state.

### **11.2 Interaction & Navigation**

a) Click to select one node; Shift/Ctrl/Cmd \+ click to select multiple nodes; verify that NODES tab in Table Panel syncs with node selection.

b) Click the background to unselect nodes; verify  that the NODES tab in Table Panel syncs and displays all data.

c) Repeat above tests for edges.

d) Pan (drag background) and zoom (wheel/trackpad); verify stability at multiple zoom levels.

e) Selection round-trip: With selected table rows, click “SELECT NODES” and verify synchronized highlights in the view.

## **12\) Tree View/Cell View (Hierarchical Networks)**

### **12.1 Toggle Views**

a) Load a hierarchical sample network (MuSIC map or U2OS map); verify tabs “Tree View” and “Cell View” appear at top of Network View.

b) Switch between Tree View and Cell View; ensure smooth transitions.

### **12.2 Interactions**

a) Single-click a system (circle) to select; confirm selection syncs to Table Panel’s NODES tab. Do this both in Tree View and Cell View.

b) Click the background to unselect nodes; verify  that the NODES tab in Table Panel syncs and displays all data. Do this both in Tree View and Cell View.

c) Shift/Ctrl/Cmd \+ click to select multiple nodes; verify that NODES tab in Table Panel syncs with node selection. Do this in Tree View only.

d) With a system selected, confirm the Sub Network Viewer (top-right toggle) pane opens correctly; confirm the underlying interaction network appears. Do this both in Tree View and Cell View.

e) With multiple systems selected, confirm the Sub Network Viewer (top-right toggle) pane opens correctly; confirm it displays the following message: “**Multiple nodes are selected. Please select one subsystem to display the associated interactions**”. Do this in Tree View only.

f) Click to select one node; Shift/Ctrl/Cmd \+ click to select multiple nodes in the Sub Network View Pane; verify that NODES tab in Table Panel syncs with node selection. Do this both in Tree View and Cell View.

g) Click the background in the Sub Network View Pane to unselect nodes; verify  that the NODES tab in Table Panel syncs and displays all data. (In Cell View only, also verify that the red highlighted gene in the system is cleared).  Do this both in Tree View and Cell View.

h) If multiple data sources exist, test the source filters below the Sub Network View pane; toggle each and confirm changes in the displayed sub network above.

i) Select Cell View; double-click to drill into sub-systems; verify zoom-in and breadcrumb/context. 

### **12.3 Share & URL**

a) From a specific cell selection/zoom, use Share Network; open the link and confirm the state persists (where supported).

## **13\) Table Panel (NODES, EDGES, NETWORK tabs)**

### **13.1 Visibility & Scope**

a) With no selection, NODES tab shows all node rows; with a selection, shows selected subset only. Repeat for EDGES.

### **13.2 Table Tools (top of panel)**

a) Search (magnifier): find a string in any column; verify filtered/highlighted results.

b) Insert New Column: create String, Boolean, Integer, Double, and List types; set optional default values.

c) Import Table from File: import a CSV to add new columns; confirm correct key mapping and data types.

### **13.3 Edit & Bulk Apply**

a) Edit individual cells (double-click); save on blur; Esc cancels during edit.

b) Select multiple cells with Shift/Cmd; verify multi-select visuals.

c) APPLY VALUE TO COLUMN: copies the selected cell’s value to the entire column; verify.

d) APPLY VALUE TO SELECTED NODES/EDGES: copies the value to only selected elements; verify.

### **13.4 Structure**

a) Move columns by dragging headers; verify new order.

b) Sort by column (ascending/descending); verify sort indicators.

c) Rename column; verify the header updates and mappings remain intact.

d) Delete Columns…; confirm irreversible removal and downstream effects in Styles.

### **13.5 Cross-highlighting**

a) Select a row; click SELECT NODES (or SELECT EDGES); verify network view selection.

## **14\) Styles (STYLE panel)**

### **14.0 General**

\- Three sub-tabs: NODES, EDGES, NETWORK.

\- For each property below, perform: (A) change Default; (B) set Discrete mapping if supported (categorical column);

(C) set Continuous mapping if supported (numeric column; color gradient for color properties); (D) set a Bypass on a selected subset and verify it overrides both Default and Mapping.

\- If a mapping type is NOT supported for a property, verify the UI does not offer it (this counts as PASS).

\- After each mapping, visually validate and also confirm via Table Panel (e.g., add temporary columns to drive mappings).

### **14.1 Node Properties (run A–D for each)**

\- Border Color (supports discrete; supports continuous color gradient).

\- Border Line Type (discrete only).

\- Border Opacity (continuous).

\- Border Width (continuous).

\- Fill Color (discrete; continuous gradient).

\- Height (continuous).

\- Label (prefer Passthrough from a text column; for discrete mapping, map categories to distinct labels; continuous not applicable—verify not offered).

\- Label Color (discrete; continuous gradient).

\- Label Font (discrete; choose among available fonts).

\- Label Font Size (continuous).

\- Label Opacity (continuous).

\- Label Position (discrete positions).

\- Label Rotation (continuous).

\- Label Width (continuous).

\- Opacity (continuous).

\- Selected Color (typically constant; try discrete by category; continuous gradient if offered; otherwise verify not offered).

\- Shape (discrete only: ellipse, rectangle, etc.).

\- Visibility (discrete boolean).

\- Width (continuous).

\- Z Order (special: default ignored; mapping must exist to take effect—verify behavior).

### **14.2 Edge Properties (run A–D for each)**

\- Label; Label Color; Label Font; Label Font Size; Label Opacity; Label Rotation; Label Width (as for nodes).

\- Line Type (discrete).

\- Opacity (continuous).

\- Selected Color (as for nodes).

\- Source Arrow Color (discrete; continuous gradient).

\- Source Arrow Shape (discrete).

\- Stroke Color (discrete; continuous gradient).

\- Target Arrow Color (discrete; continuous gradient).

\- Target Arrow Shape (discrete).

\- Visible (discrete boolean).

\- Width (continuous).

\- Source Arrow Selected Paint; Stroke Color (Selected); Color (Unselected) (treat as colors; verify which mapping types are offered).

\- Z Order (as for nodes).

### **14.3 Network Properties**

\- Background Color (set Default; test discrete mapping by category if offered; continuous gradient if offered; otherwise verify not offered).

### **14.4 Style Robustness & Persistence**

\- Save/Reload: Save network/workspace; reload and verify styles persist.

\- Interaction with Deletes: After deleting columns used by mappings, confirm style indicates missing mappings gracefully.

\- Performance: Apply continuous mapping to a large numeric column; verify interactivity remains acceptable.

## **15\) URL Query Parameters (Advanced)**

### **15.1 Import via URL**

\- Open Cytoscape Web using ?import=\<CX2 URL\> (use a locally served or accessible URL). Verify auto-load.

### **15.2 State Parameters**

\- Construct a URL with selectednodes, selectededges, left/right/bottom (open/closed), activeTableBrowserTab (0/1/2).

\- Open it in a new tab and verify selections and panel states.

### **15.3 Share Network**

\- Use Share Network tool on a customized state; validate the link reproduces the state as expected.

## **16\) Troubleshooting & Recovery**

### **16.1 Hard Reload**

\- Perform Shift+Reload; confirm the app reloads and stale cache is bypassed.

### **16.2 Clear Browser Cache**

\- Clear cache per browser docs; reopen Cytoscape Web; confirm normal operation resumes.

### **16.3 Clear Local Database**

\- As a last resort, Data → Clear Local Database; confirm the environment resets to a clean state.

### **16.4 Bug Reporting**

\- Help → Report a Bug; verify link works and include environment notes gathered above.
