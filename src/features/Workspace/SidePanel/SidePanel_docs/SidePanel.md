# SidePanel Component

## Overview

`SidePanel` is a collapsible side panel that displays additional UI components and the Sub Network Viewer. It provides tab navigation between different panels and handles network activation when switching to the Sub Network Viewer tab.

## Behavior

### Tab Navigation

- Displays tabs for available panels, including the "Sub Network Viewer" and dynamically loaded app panels
- Uses MUI `Tabs` component with scrollable variant for tab navigation
- Tab selection is managed via local `value` state
- Tab contents are generated by `getTabContents()` function

### Network Activation

When the "Sub Network Viewer" tab (index 0) is clicked, the component activates the appropriate network view:

**Activation Logic:**
1. **If `activeNetworkView` is already an interaction network**: Uses it directly
2. **Looks for interaction networks**: Searches the networks store for interaction networks (IDs containing `_`) that are related to the current network (start with `currentNetworkId_`)
3. **Falls back to `activeNetworkView`**: If it exists and is not empty
4. **Finally falls back to `currentNetworkId`**: The main network ID

**Why this logic?**
- The Sub Network Viewer displays subnetworks (interaction networks) that have IDs like `hierarchyId_subsystemId`
- When clicking the tab, we need to activate the subnetwork being displayed, not the main hierarchy network
- The logic prioritizes finding the actual subnetwork ID over using the main network ID

**Handlers:**
- **`onChange` handler**: Called when switching tabs. Activates network view if switching to "Sub Network Viewer" tab.
- **`onClick` handler on each Tab**: Handles the case where the "Sub Network Viewer" tab is already selected (since `onChange` doesn't fire). Ensures network activation happens even when clicking an already-selected tab.

## Component Structure

```
SidePanel
├── Tabs (MUI component)
│   └── Tab[] (one per panel)
│       └── onClick handler (activates network view for Sub Network Viewer tab)
└── Box (content area)
    └── TabPanel[] (one per panel, only selected one is visible)
        └── ViewerPanel (for Sub Network Viewer)
        └── AppPanel[] (for dynamically loaded app panels)
```

## Integration Points

- **WorkspaceStore**: Gets `currentNetworkId` to determine the main network
- **UiStateStore**: 
  - Reads `activeNetworkView` to check current active network
  - Updates `activeNetworkView` to activate network views
- **NetworkStore**: Searches for interaction networks (subnetworks) in the networks store
- **AppStore**: Gets active app components to display as panels

## Helper Functions

### `isSubnetwork(networkId: IdType): boolean`

Checks if a network ID represents a subnetwork by checking if it contains an underscore. Subnetworks have the format `hierarchyId_subsystemId`. This function is imported from `HierarchyViewer/utils/hierarchyUtil`.

### `getNetworkIdToActivate(): IdType | null`

Determines which network ID should be activated when clicking the "Sub Network Viewer" tab. Uses a priority-based approach:
1. Active interaction network (if already set)
2. Related interaction networks from the store
3. Current active network view
4. Current network ID

## Design Decisions

**Network Activation on Tab Click**: When clicking the "Sub Network Viewer" tab, the component actively searches for and activates the appropriate subnetwork. This ensures that the network view is properly activated even if the user is switching from another tab or clicking an already-selected tab.

**Interaction Network Detection**: Uses the underscore pattern (`_`) to identify subnetworks. This is a convention used throughout the codebase for interaction networks.

**Fallback Logic**: The multi-level fallback ensures that a network is always activated when possible, even if the exact subnetwork isn't found in the store yet.

## Related Components

- **TabContents**: Generates the tab panel contents, including the "Sub Network Viewer" and app panels
- **TabPanel**: Renders individual tab panel content
- **ViewerPanel**: The HierarchyViewer component displayed in the "Sub Network Viewer" tab
- **NetworkPanel**: Receives activation updates and manages the active state of network views
