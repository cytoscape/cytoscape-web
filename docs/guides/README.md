# Developer Guides

> Practical guides written for human developers working on Cytoscape Web.

## Purpose

This directory contains how-to guides, tutorials, and onboarding materials aimed at human contributors. Unlike `design/` (system internals) or `specifications/` (behavioral rules), guides here focus on **getting things done** — step-by-step instructions for common development tasks.

## Scope

- Local development setup and troubleshooting
- How to add a new feature module, store, or model
- Testing workflows (unit tests, E2E tests, fixtures)
- Debugging tips and common pitfalls
- Deployment and release procedures
- Contributing guidelines and code review practices

## File Naming

```
<topic>.md
```

- Use kebab-case (e.g., `adding-a-new-store.md`, `local-dev-setup.md`)
- One guide per topic — keep each document focused and self-contained

## Writing Guidelines

- Write for a developer who is new to the project
- Use numbered steps for procedures
- Include terminal commands as fenced code blocks
- Link to relevant source files and other docs when helpful
- Keep language concise — prefer short paragraphs and bullet lists
